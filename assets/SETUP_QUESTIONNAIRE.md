# 使用者首次设置问卷

请使用者填写以下内容。答案可以直接写入 `config.yaml`，不需要写程序。

## A. 数据来源

1. 聊天记录所在路径：
2. 文件类型：TXT / JSON / HTML / CSV / 其他
3. 是否包含子目录：是 / 否
4. 数据是否由本机应用持续更新：是 / 否

## B. 群组范围

1. 群组标题必须包含哪些字样？
   - 示例：`XXXXXX`
2. 多个字样是“任一符合”还是“全部符合”？
3. 哪些群组标题必须排除？
4. 是否限制特定发送者？
5. 是否限制消息内容关键词？

## C. 时间范围

1. 首次运行检查过去多少小时？
2. 后续是否从上次成功执行时间继续？
3. 是否需要重叠检查最近若干分钟，以避免遗漏延迟消息？

## D. 需求类别

1. 使用默认类别吗？
   - New Feature
   - Improvement
   - Bug / Debug
2. 是否增加其他类别？
3. 哪些类别允许自动写入？
4. 哪些类别必须人工确认？

## E. 历史资料

1. 历史日报存放在哪里？
2. 是否有既有需求清单导出文件？
3. 是否已建立本地需求索引？
4. 哪些资料禁止上传到公开仓库？

## F. 目标项目系统

1. 目标系统名称：
2. 项目链接或项目识别信息：
3. 访问方式：连接器 / API / CLI / 浏览器
4. 是否需要首次人工登录？
5. 是否允许同一执行环境复用登录状态？

## G. 字段映射

请填写目标系统的真实字段名称：

- 标题：
- 描述：
- 类型：
- 优先级：
- 来源群组：
- 来源时间：
- 原始上下文：
- 验收条件：
- 相关既有事项：

## H. 写入规则

1. 首次运行只预览还是允许写入？
2. 是否只写入确认全新的事项？
3. 相似事项是否只放入日报？
4. 既有事项补充是否允许自动更新？
5. 遇到不明确结果时是否停止写入？

## I. 输出

1. 日报输出目录：
2. 日报格式：Markdown / CSV / 其他
3. 是否需要隐藏发送者姓名？
4. 原始上下文最多保留多少？
5. 失败时希望收到哪些信息？
